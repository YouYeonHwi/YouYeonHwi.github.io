## 1. 튜플(tuple)이란?

튜플(`tuple`)은 리스트와 매우 유사하지만, 한 번 생성되면 **그 내용을 수정할 수 없는(immutable)** 자료형입니다. 리스트가 대괄호 `[]`를 사용하는 것과 달리, 튜플은 소괄호 `()`를 사용하여 표현합니다.

- **순서(Sequence)**: 데이터가 저장된 순서가 그대로 유지됩니다.
- **불변성(Immutable)**: 생성 후 요소의 추가, 수정, 삭제가 불가능합니다. 이로 인해 데이터의 무결성을 보장해야 할 때 유용합니다.
- **중복 허용**: 리스트와 마찬가지로 동일한 값을 여러 번 저장할 수 있습니다.

## 2. 튜플 생성하기

```python
# 1. 소괄호 () 사용
t1 = (1, 2, 3, 4, 5)
t2 = ("a", "b", "c")
t3 = (1, "hello", True)

# 2. 소괄호 생략 가능 (패킹)
t4 = 1, 2, 3  # 이 역시 튜플입니다. (1, 2, 3)과 동일
print(type(t4)) # <class 'tuple'>

# 3. 요소가 하나인 튜플 생성 시 주의점
# (1)은 그냥 숫자 1입니다. 튜플로 인식시키려면 뒤에 쉼표(,)를 붙여야 합니다.
single_element_tuple = (1,)
not_a_tuple = (1)
print(type(single_element_tuple)) # <class 'tuple'>
print(type(not_a_tuple))      # <class 'int'>

# 4. tuple() 생성자 사용
t5 = tuple(range(5))
print(t5) # (0, 1, 2, 3, 4)
```

## 3. 튜플은 왜 사용할까?

리스트가 있는데도 튜플을 사용하는 주된 이유는 다음과 같습니다.

1.  **데이터의 무결성 보장**: 프로그램이 실행되는 동안 변경되어서는 안 되는 중요한 데이터를 안전하게 보호할 수 있습니다. 예를 들어, 함수의 인자로 전달되거나 반환 값으로 사용될 때 데이터가 예기치 않게 변경되는 것을 막아줍니다.

2.  **딕셔너리의 키(Key)로 사용 가능**: 딕셔너리의 키는 변하지 않는(immutable) 값이어야 합니다. 따라서 가변적인 리스트는 키가 될 수 없지만, 불변인 튜플은 키로 사용할 수 있습니다.

    ```python
    locations = {
        (37.56, 126.97): "서울",
        (35.17, 129.07): "부산"
    }
    print(locations[(37.56, 126.97)]) # "서울"
    ```

3.  **약간의 성능상 이점**: 일반적으로 튜플은 리스트보다 메모리를 적게 사용하고, 처리 속도도 약간 더 빠릅니다. 아주 큰 데이터를 다룰 때 미미한 성능 향상을 기대할 수 있습니다.

## 4. 패킹과 언패킹 (Packing and Unpacking)

튜플의 가장 강력하고 편리한 기능 중 하나는 패킹과 언패킹입니다.

- **패킹(Packing)**: 여러 개의 값을 쉼표로 구분하여 하나의 변수(튜플)에 담는 과정입니다.
- **언패킹(Unpacking)**: 튜플의 각 요소를 여러 개의 변수에 나누어 할당하는 과정입니다.

```python
# 패킹 (Packing)
my_tuple = "Alice", 25, "Engineer"  # 3개의 값이 my_tuple 하나에 패킹됨
print(my_tuple) # ('Alice', 25, 'Engineer')

# 언패킹 (Unpacking)
# 튜플의 요소 개수와 변수의 개수가 정확히 일치해야 합니다.
name, age, job = my_tuple
print(f"Name: {name}, Age: {age}, Job: {job}")
# Name: Alice, Age: 25, Job: Engineer
```

언패킹은 변수 값을 교환(swap)할 때도 유용하게 사용됩니다.

```python
a = 10
b = 20

# 임시 변수 없이 두 변수의 값을 교환
a, b = b, a

print(f"a: {a}, b: {b}") # a: 20, b: 10
```

## 5. 튜플의 연산과 메서드

튜플은 불변이기 때문에 요소를 변경하는 메서드(`append`, `remove` 등)는 없지만, 새로운 튜플을 만드는 연산이나 정보를 얻는 메서드는 사용할 수 있습니다.

- **인덱싱과 슬라이싱**: 리스트와 동일하게 사용 가능합니다.
- **`+` 연산**: 두 튜플을 합쳐 새로운 튜플을 만듭니다.
- **`*` 연산**: 튜플을 여러 번 반복하여 새로운 튜플을 만듭니다.
- **`len(t)`**: 튜플 `t`의 길이를 반환합니다.
- **`t.count(값)`**: 튜플에 특정 값이 몇 개 있는지 셉니다.
- **`t.index(값)`**: 특정 값의 첫 번째 위치(인덱스)를 반환합니다.

```python
t1 = (1, 2, 3)
t2 = (4, 5)

# 덧셈과 곱셈
print(t1 + t2) # (1, 2, 3, 4, 5)
print(t2 * 3)  # (4, 5, 4, 5, 4, 5)

# 인덱싱과 슬라이싱
t3 = (10, 20, 30, 40, 50)
print(t3[1])       # 20
print(t3[2:])      # (30, 40, 50)

# count와 index
t4 = (1, 2, 1, 3, 1, 4)
print(f"1의 개수: {t4.count(1)}") # 1의 개수: 3
print(f"값 3의 위치: {t4.index(3)}") # 값 3의 위치: 3
```

튜플은 '수정할 수 없는 리스트'로 단순하게 생각할 수도 있지만, 그 불변성이라는 특징이 프로그램의 안정성과 효율성을 높여주는 중요한 역할을 합니다. 데이터를 안전하게 보호하고 싶을 때, 튜플 사용을 적극적으로 고려해 보세요.
