## 1. 모듈(Module)이란 무엇일까?

모듈은 **관련 있는 변수, 함수, 클래스 등을 모아놓은 하나의 파이썬 파일(`.py`)**입니다. 코드가 길어지고 복잡해질 때, 관련된 기능들을 별도의 파일로 분리하여 코드의 재사용성을 높이고, 유지보수를 용이하게 만들어주는 중요한 개념입니다.

우리가 레고 블록으로 큰 성을 만든다고 상상해봅시다. 성벽, 탑, 문 등 각 부분을 미리 작은 단위로 만들어두고, 필요할 때 가져와 조립하면 훨씬 효율적일 것입니다. 여기서 각 '부분'에 해당하는 것이 바로 '모듈'입니다.

**모듈을 사용하는 이유:**
- **이름공간(Namespace) 분리**: 모듈마다 독립된 이름공간을 가지므로, 서로 다른 모듈에 동일한 이름의 함수나 변수가 있어도 충돌하지 않습니다.
- **코드의 재사용**: 자주 사용되는 함수나 클래스를 모듈로 만들어두면, 어떤 프로그램에서든 쉽게 불러와 재사용할 수 있습니다.
- **관리의 용이성**: 거대한 하나의 파일 대신, 기능별로 여러 개의 작은 파일로 코드를 나누어 관리하므로, 특정 기능을 찾거나 수정하기가 훨씬 수월해집니다.

## 2. 모듈 생성 및 사용

### 1. 모듈 생성하기

모듈을 만드는 것은 매우 간단합니다. 그냥 일반적인 파이썬 코드를 담은 `.py` 파일을 만들기만 하면 됩니다.

**`my_math.py` (모듈 파일 예시)**
```python
# my_math.py

# 변수
PI = 3.14159

# 함수
def add(a, b):
    """두 수를 더한 값을 반환합니다."""
    return a + b

def subtract(a, b):
    """두 수를 뺀 값을 반환합니다."""
    return a - b

# 클래스
class Circle:
    """원의 넓이를 계산하는 클래스"""
    def __init__(self, radius):
        self.radius = radius

    def get_area(self):
        return PI * (self.radius ** 2)
```
> 위와 같이 `my_math.py` 파일을 저장하면, 이 파일 자체가 `my_math`라는 이름의 모듈이 됩니다.

### 2. 모듈 불러오기: `import`

다른 파이썬 파일에서 `my_math` 모듈의 기능을 사용하려면 `import` 문을 사용해야 합니다.

**`main.py` (모듈을 사용하는 파일)**
```python
# main.py

# 1. import 모듈이름
# 모듈 전체를 가져옵니다. 모듈 안의 함수나 변수를 사용하려면 '모듈이름.변수' 형식으로 접근해야 합니다.
import my_math

result_add = my_math.add(10, 5)
print(f"덧셈 결과: {result_add}")  # 덧셈 결과: 15

print(f"원주율: {my_math.PI}")   # 원주율: 3.14159

my_circle = my_math.Circle(10)
print(f"반지름이 10인 원의 넓이: {my_circle.get_area()}") # 반지름이 10인 원의 넓이: 314.159

# -----------------------------------------------------------

# 2. from 모듈이름 import 가져올_항목
# 모듈에서 특정 함수, 변수, 클래스만 골라서 가져옵니다.
# 이렇게 가져온 항목은 '모듈이름.' 없이 바로 사용할 수 있습니다.
from my_math import subtract, Circle

result_subtract = subtract(10, 5) # 'my_math.' 없이 바로 사용
print(f"뺄셈 결과: {result_subtract}") # 뺄셈 결과: 5

your_circle = Circle(5) # 'my_math.' 없이 바로 사용
print(f"반지름이 5인 원의 넓이: {your_circle.get_area()}")

# -----------------------------------------------------------

# 3. from 모듈이름 import *
# 모듈 안의 모든 항목(*)을 가져옵니다.
# 편리해 보이지만, 어떤 항목이 현재 이름공간으로 들어왔는지 불분명해지고,
# 기존에 있던 이름과 충돌할 위험이 있어 권장되지 않는 방법입니다.
# from my_math import *
# print(add(100, 200)) # 'my_math.' 없이 사용 가능

# -----------------------------------------------------------

# 4. import 모듈이름 as 별명
# 모듈 이름이 너무 길거나 다른 이름과 겹칠 때 별명을 붙여 사용할 수 있습니다.
import my_math as mm

result_add_alias = mm.add(7, 3)
print(f"별명을 사용한 덧셈 결과: {result_add_alias}") # 10

from my_math import subtract as sub
result_sub_alias = sub(7, 3)
print(f"별명을 사용한 뺄셈 결과: {result_sub_alias}") # 4
```

## 3. `if __name__ == "__main__"`의 의미 (꿀팁!)

파이썬 파일을 실행하는 방식은 두 가지입니다.
1.  **직접 실행**: `python my_math.py` 처럼 터미널에서 직접 해당 파일을 실행하는 경우.
2.  **모듈로 임포트**: 다른 파일에서 `import my_math` 처럼 모듈로 불러오는 경우.

파이썬 인터프리터는 코드를 실행하기 전에 몇 가지 특별한 변수를 설정합니다. 그중 하나인 `__name__` 변수는,
- 파일이 **직접 실행**될 때 `__main__` 이라는 값을 가집니다.
- 파일이 **모듈로 임포트**될 때 해당 **모듈의 이름**(`my_math`) 값을 가집니다.

이 특징을 이용하여, 특정 코드가 **파일이 직접 실행될 때만 작동하도록** 만들 수 있습니다. 이는 주로 해당 모듈을 테스트하기 위한 코드를 작성할 때 매우 유용합니다.

**`my_math.py` (테스트 코드 추가)**
```python
# my_math.py

# ... (기존 변수, 함수, 클래스 정의) ...

# 이 파일이 'python my_math.py'로 직접 실행될 때만 아래 코드가 동작함
if __name__ == "__main__":
    print("my_math 모듈을 직접 실행했습니다. 자체 테스트를 시작합니다.")
    
    test_add = add(5, 5)
    print(f"add(5, 5) 테스트: {test_add}")

    test_circle = Circle(2)
    print(f"Circle(2) 넓이 테스트: {test_circle.get_area()}")

    print("테스트 완료.")

# 'main.py'에서 'import my_math'로 이 파일을 임포트할 때는 __name__이 "my_math"가 되므로,
# 위의 if 블록 안의 코드는 실행되지 않습니다.
```

**실행 결과:**

1.  `python my_math.py` 실행 시:
    ```
    my_math 모듈을 직접 실행했습니다. 자체 테스트를 시작합니다.
    add(5, 5) 테스트: 10
    Circle(2) 넓이 테스트: 12.56636
    테스트 완료.
    ```

2.  `python main.py` 실행 시: (`my_math` 모듈의 테스트 코드는 출력되지 않음)
    ```
    덧셈 결과: 15
    원주율: 3.14159
    반지름이 10인 원의 넓이: 314.159
    ...
    ```
> 이처럼 `if __name__ == "__main__"` 구문은 모듈의 독립적인 테스트 기능을 제공하여 개발과 디버깅을 훨씬 편리하게 만들어주는 필수적인 코드 패턴입니다.

## 4. 모듈 검색 경로 (Module Search Path)

파이썬이 `import my_module` 문을 만났을 때, 어떤 순서로 `my_module.py` 파일을 찾을까요? 파이썬은 정해진 경로들을 순서대로 확인합니다.

1.  **현재 디렉터리**: 스크립트가 실행되고 있는 현재 폴더.
2.  **PYTHONPATH 환경 변수**: 셸의 `PATH` 환경 변수처럼, 모듈을 찾을 추가적인 디렉터리 경로들을 지정해 놓은 환경 변수.
3.  **표준 라이브러리 디렉터리**: 파이썬이 설치될 때 기본적으로 함께 설치되는 모듈들이 있는 디렉터리.

`sys` 모듈을 이용해 현재 파이썬의 모듈 검색 경로를 확인할 수 있습니다.

```python
import sys

print(sys.path)
# 출력 (예시):
# [
#   'd:\my_project',  # 현재 디렉터리
#   'C:\Python39\lib\site-packages', # pip로 설치한 패키지들
#   'C:\Python39\lib', # 표준 라이브러리
#   ...
# ]
```

만약 직접 만든 모듈이 이 경로들 안에 없다면 `ModuleNotFoundError`가 발생합니다. 이 문제를 해결하려면 모듈을 검색 경로 안으로 옮기거나, `sys.path.append()`를 이용해 해당 모듈이 있는 경로를 동적으로 추가할 수 있습니다.

모듈은 파이썬 프로그래밍의 확장성과 체계성을 책임지는 핵심 요소입니다. 표준 라이브러리 모듈부터 외부 라이브러리, 그리고 직접 만드는 모듈까지, 모듈의 원리를 잘 이해하고 활용하는 것이 중요합니다.
